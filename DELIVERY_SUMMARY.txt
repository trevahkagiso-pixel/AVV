USAGE EXAMPLES
==============

Example 1: Run Single Backtest
────────────────────────────────
from ichimoku_strategy import create_ichimoku_strategy
from backtest_runner import run_backtest_with_custom_strategy

strategy = create_ichimoku_strategy()
stats, df, bt = run_backtest_with_custom_strategy('EUR_USD_daily', strategy)


Example 2: Test All Pairs
──────────────────────────
from rsi_strategy import create_rsi_strategy
from backtest_runner import run_all_pairs_with_strategy

strategy = create_rsi_strategy()
summary = run_all_pairs_with_strategy(strategy)


Example 3: Compare Strategies
──────────────────────────────
from backtest_runner import run_multiple_strategies

results = run_multiple_strategies('EUR_USD_daily', {
    'ichimoku': create_ichimoku_strategy(),
    'rsi': create_rsi_strategy(),
})


Example 4: Add New Strategy (5 Lines!)
───────────────────────────────────────
class MyStrategy(BaseStrategy):
    def add_indicators(self, df): return df
    def generate_signals(self, df): df['signal']=0; return df

strategy = MyStrategy()
stats, df, bt = run_backtest_with_custom_strategy('EUR_USD_daily', strategy)

DESIGN PRINCIPLES IMPLEMENTED
==============================

1. SOLID Principles
   ✓ Single Responsibility - Each class has one purpose
   ✓ Open/Closed - Open for extension, closed for modification
   ✓ Liskov Substitution - All strategies implement same interface
   ✓ Interface Segregation - Minimal required methods
   ✓ Dependency Inversion - Depends on abstractions, not concrete classes

2. Clean Code
   ✓ Meaningful names
   ✓ Small focused methods
   ✓ DRY (Don't Repeat Yourself)
   ✓ Clear separation of concerns
   ✓ Comprehensive documentation

3. Design Patterns
   ✓ Strategy Pattern - Core to the framework
   ✓ Registry Pattern - For strategy management
   ✓ Factory Pattern - Easy strategy creation
   ✓ Template Method - Base class provides structure
